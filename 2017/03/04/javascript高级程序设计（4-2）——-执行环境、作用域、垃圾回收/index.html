javascript高级程序设计（4.2）—— 执行环境、作用域、垃圾回收

标签（空格分隔）：javascript

拾遗

执行环境及作用域

作用域链

  • 代码在一个环境中执行的时候，会生成一个作用域链，使得执行环境有权访问的所有变量和函数可以有序访问。

  • 要延长作用域链

    • try-catch语句的catch块，catch(err){}，err是个错误对象
    • with语句

没有块级作用域

  • if和for语句虽然有花括号包围，但是没有块状作用域，因此在这两个语句中声明的变量会添加到当前的执行环境中，而不是像其他语言在语句结束后将变量销毁

  • 延长作用域链并不是新增一个作用域，只是说可以直接访问这个变量对象中的属性，而由于没有块级作用域，在with函数中声明的变量是加到当前的执行环境中的。

垃圾回收

标记清除

  • 标记和收集
    • 标记阶段会给存储在内存中的所有变量都加上标记
    • 去掉环境中的变量以及被环境中的变量引用的变量的标记
    • 之后在离开环境之后，将在此环境的变量做上标记，等待被回收

引用计数（因为存在循环引用的问题，基本都使用标记清除的算法）

  • 当声明一个变量（比如一个对象）并将一个引用类型的值赋给该变量时，这个值的引用次数就是1，如果同一个值被赋给另一个变量，那么这个值的引用次数加1

  • 如果包含该值的引用变量取得了另一个值，则这个值次数减1，当这个引用值变为0时，被回收

  • 在以前，BOM和DOM都不是js对象，他们垃圾回收的机制是引用计数，不过IE9以后把BOM和DOM都转换为真正的js对象，避免了两种垃圾收集算法并存所导致的问题

解除引用

  • 可以手动将对象设置为null，使它立刻脱离执行环境，方便垃圾收集器下次执行时将其回收
  • 手动解除引用是解决循环引用的好方法（虽然现在这种垃圾回收算法不再出现在现代浏览器中）
