javascript高级程序设计（6.1）—— 理解对象、创建对象

标签（空格分隔）：javascript

理解对象

对象属性类型

数据属性（data descriptor）

  • [[configurable]]：表示能否通过delete删除属性，能否修改属性的特性，或者能否把属性修改为访问其属性
    • configurable设置为false后，这4个数据属性只能将writable设置为false，其他的设置操作将报”Cannot redefine property”的错误，也就是说也不能再将自己设为true
  • [[enumerable]]：表示能否通过for-in循环返回属性
  • [[writable]]：表示能否用第一种方式修改属性的值
    • 也就是说就算writable被设置为false，也可以通过Object.defineProperty()方法来修改value属性从而达到修改值的效果
    • 当然如果configurable为false，writable也为false，那这个属性的值将永远不可改变，当然你也别想删掉
  • [[value]]：包含这个属性的数据值。默认为undefined

访问器属性（accessor descriptor）

访问器属性可以理解为，想通过person.c获取一个值，但是c并不是person对象的一个属性，那么可以通过设置一个访问器，配制get和set函数来达到访问属性和修改属性的目的

  • [[configurable]]：同数据属性
  • [[enumerable]]：同数据属性
  • [[get]]：在读取属性时调用的函数。默认值为undefined
    • 若不设置此函数，则视为该属性不可读。直接访问该属性会返回undefined，严格模式下则报错
  • [[set]]：在写入属性时调用的函数。默认指为undefined
    • 若不设置此函数，则视为该属性不可写。尝试写入属性会被忽略，严格模式下则报错

定义多个属性

Object.defineProperties()

读取属性的特性

Object.getOwnPropertyDescriptor()

创建对象

工厂模式

  • 工厂模式不能解决对象识别的问题（及怎样知道一个对象的类型）

构造函数模式

new操作符

  • 创建一个新对象
  • 将构造函数的作用域赋给新对象
  • 执行构造函数中的代码
  • 返回新对象

该模式特点

  • person1和person2是Person不同的实例
  • 如果不使用new操作符直接调用该函数，那么this对象指向Gloabal对象（浏览器中是window对象）
  • person1和person2的sayName()方法不是同一个方法，我们每当创建一个实例就要新增一个sayName()方法，这是没有必要的
  • 虽然这样确实只有一个方法了，但是放在全局中就没有封装性可言了

原型模式

每个函数都有一个prototype（原型）属性，这个属性是个指针，指向一个对象，可以包含特定类型的所有实例共享的属性和方法

理解原型对象

  • 构造函数Person
    • Person.prototype指向Person的原型对象Person Prototype
  • 原型对象Person Prototype
    • 原型对象的constructor指向构造函数Person
  • 实例person1
    • 实例中的[[Prototype]]指向原型对象
    • 对象上的__proto__属性，一般对脚本不可见
  • 当代码读取某个对象的属性时
    • 先从对象实例开始，若找到则返回该属性的值
    • 若没有找到，则从原型对象中查找给定名字的属性

原型属性存在与可枚举方法总结

注意之后的代码都是连续的，前后相关的，请注意属性的变化

hasOwnProperty()
in操作符
Object.getOwnPropertyNames()
  • 所有的在实例中的属性，不管可不可以枚举，都会获取到
  • 在原型中的属性，不可以获取到
Object.keys()
  • 所有的在实例中的可枚举的属性，才会获取到
propertyIsEnumerable()
  • 所有的在实例中的可枚举的属性，才会是true
for-in循环
  • 返回的是所有能够通过对象访问的、可枚举的属性，包含实例中的属性和原型中的属性
  • 如果实例中的可枚举属性屏蔽了原型中的不可枚举属性（表明属性同名），那么也会出现在for-in循环中（IE8及以前会有bug）
  • 如果实例中的不可枚举属性屏蔽了原型中的可枚举属性（表明属性同名），那么也会出现在for-in循环中（这条用代码测出来的，见上，暂不能理解为什么）

组合使用构造函数和原型模式

动态原型模式

  • 由于new操作符会先创建对象再执行构造函数，所以不能直接重写Person.prototype

寄生构造函数模式（不常用）

  • 个人感觉不如用继承，但可能还没讲到继承
  • 不能依赖instanceof操作符来确定对象类型

稳妥构造函数模式（不常用）

  • 保证除了sayName()函数以外，不可能有别的方法访问传入到构造函数中的原始数据
