javascript高级程序设计（7.1）—— 递归、闭包

标签（空格分隔）： javascript

函数表达式

函数声明提升

以函数声明的形式定义的函数，调用语句可以在声明语句之前

递归

  • 使用固定函数名的递归会导致在函数被其他指针引用的时候出问题（当之前的指针设为null之后，调用新指针对应的函数时会出问题）
  • 之前使用arguments.callee的方案在严格模式下会出现问题
  • 可以使用命名函数表达式达到最好效果

闭包

闭包是指有权利访问另一个函数作用域中变量的函数

找的资料

  • 学习Javascript闭包（Closure）—— 阮一峰的日志
  • js 中的活动对象 与 变量对象 什么区别？——毕加锁的回答

函数从创建到执行到销毁的流程

  1. 全局环境的变量对象始终存在
  2. 创建函数时，会创建一个预先包含全局变量对象的作用域链，这个作用域链保存到内部的[[scope]]属性中，并且把外部函数的活动对象推入到它的作用域链的前端
  3. 函数执行时，会先创建一个新的执行环境，通过复制函数的[[scope]]属性，将它添加到执行环境的作用域链中，然后又一个活动对象创建并推入执行环境作用域的前端
  4. 当在函数中访问一个变量的时候，就会从作用域链中搜索具有相应名字的变量
  5. 函数执行完毕后，局部活动变量会被销毁，内存中仅保存全局作用域，但闭包的情况有所不同

答案是10，原因是当var foo1 = foo;语句执行的时候，将foo1引用到名为foo的函数，所以[[scope]]属性，是个作用域链，当中只有全局的变量对象，当foo1执行时，活动变量中其实是foo中的变量（只用arguments），并将其推入到执行环境的作用域链中，执行打印x的操作当然找到的x是全局中的，所以结果为10，注意：只有当新的函数声明时，才会将父级函数的活动变量加入到它的作用域链中（[[scope]]属性）

名词对比

变量对象 vs. 活动对象

可以理解成，活动对象是当前激活的变量对象，只有在函数执行时，函数的变量对象才会“激活”变成活动对象并添加到执行环境的作用域链中

执行环境 vs. 活动对象

执行环境中作用域链会包含当前的活动对象，而且函数执行完毕后，执行环境会被销毁，活动对象不一定会被销毁（可能存在闭包）

关于this对象

  • 匿名函数的执行环境具有全局性
  • 每个函数在被调用时都会自动取得两个特殊变量：this和arguments，这两个属性都只会搜索到当前的活动对象，不可能直接访问外部函数的这两个值

内存泄漏

闭包在一些IE版本的中会导致一些特殊的问题（dom元素和js中的变量回收机制不同，遇到循环引用时，该元素就无法被销毁）

解决方案

  • 通过把element.id的一个副本保存在一个变量中，并且在闭包中引用该变量消除了循环引用，但还不足以解决内存泄漏。
  • 闭包会引用包含函数的整个活动对象，而其中包含着element。即使闭包不直接引用element，包含函数的活动对象中也仍然会保存一个引用。因此，有必要把element变量设置为null。
  • 这样就能解除对dom对象的引用，确保正常回收其占用的内存
