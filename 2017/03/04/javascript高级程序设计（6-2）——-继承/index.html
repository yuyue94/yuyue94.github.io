javascript高级程序设计（6.2）—— 继承

标签（空格分隔）：javascript

继承

原型链

特点

  • 实现继承的基本方法是将子类的原型设置为父类原型的实例
    • 子类原型的constructor不再指向子类的构造函数，因为原型被设置成了新的对象，当获取constructor的时候，获取到的是父类的构造函数
    • 子类的实例的调用方法，获取属性都会是
      1. 从实例中寻找
      2. 从原型中寻找
      3. 从原型的原型中寻找（因为子类的原型是父类的实例）
      4. 依次直到最顶层的父类的原型
  • 子类的实例使用instanceof操作符
  • 先讲子类的原型设置为父类原型的实例，再给子类原型添加方法（赋值会让原型指向新的对象）
  • 也不能使用字面量给子类添加新的方法

问题

  • 之前最好的创建对象的方法是组合使用构造函数模式和原型模式，思路是将实例属性放到构造函数中，方法放到原型中，但在纯原型链中会出问题
  • 由于SubType的原型是SuperType的实例，所以SubType的原型中有colors这个属性，是个引用类型，会导致它的所有实例的colors属性共享
  • 创建子类型的实例时，不能向超类型的构造函数中传递参数
